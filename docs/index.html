🚀 快速开始

4 步完成从 Fork 到在线编辑

1

Fork 仓库

点击 GitHub 页面右上角 Fork 按钮，将仓库复制到你的账号下

2

启用 GitHub Pages

进入仓库 Settings → Pages → Source 选择 GitHub Actions

3

打开在线编辑器

访问 https://islinxu.github.io/llm-resume-template/editor.html

4

编辑并构建

在编辑器中修改个人信息和内容，点击「🚀 构建」提交到 GitHub，CI 自动编译并部署 PDF

📂 项目架构

三层分离，修改互不干扰

latex_resume_pro/ ├── resume-pro.cls # 核心类文件（763 行） ├── resume-pro-zh.tex # 中文主入口 ├── config.tex # 全局配置（主题/个人信息/Logo） ├── modules/ │ ├── header.tex # 头部 + 摘要 │ ├── skills.tex # 专业技能 │ ├── education.tex # 教育背景 │ ├── experience.tex # 实习/工作/科研 │ ├── projects.tex # 项目 + 开源 │ └── publications.tex # 论文/专利/荣誉 ├── variants/ # 11 种场景变体 ├── logos/ # 73 个 Logo 占位符 └── fonts/ # 字体回退链配置

❓ 常见问题

使用过程中的常见疑问解答

需要安装 LaTeX 环境吗？▼
不需要！使用在线编辑器 + GitHub Actions CI，所有编译都在云端完成。如果你想本地编译，可以使用 Docker：docker compose build
如何更换主题颜色？▼
在编辑器的「🎨 主题」面板中选择 8 种预设主题之一，或者使用「自定义主题」输入任意 HEX 色值。也可以直接修改 config.tex 中的 \\settheme{} 或 \\customtheme{} 命令。
如何添加自己的 Logo？▼
将 PNG 格式的 Logo 文件放入 latex_resume_pro/logos/ 目录，然后在 config.tex 中定义短宏：\\def\\myCompanyLogo{logos/my-company.png}，之后在模块中使用 \\experienceItem[\\myCompanyLogo]{...} 引用。
如何制作一页简历？▼
选择「一页紧凑」变体，或在 config.tex 中添加 \\setMaxPages{1}。同时可以在编辑器的「编译选项」中选择「紧凑」模式和「隐藏页脚」，减少空间浪费。
如何制作英文简历？▼
选择「英文版」变体，或在主文件中将 \\documentclass[zh] 改为 \\documentclass[en]。同时需要将 modules/ 下的内容替换为英文版本（参考 modules-en/ 目录）。
GitHub Token 有什么权限要求？▼
Token 需要 repo 权限（Full control of private repositories），这样才能通过 API 提交文件到仓库。建议使用 Fine-grained token，仅授予目标仓库的 Contents 读写权限。
如何自定义字体？▼
字体回退链在 resume-pro.cls 中定义：IBM Plex Sans → Noto Sans CJK SC → PingFang SC → Source Han Serif SC。你可以修改 cls 文件中的字体声明来使用自己的字体。
